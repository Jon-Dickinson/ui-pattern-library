The float property converts inline-block to block

- are the properties of block added during this process?

If we consider the following HTML element:

<div></div>

Initial base styling is added by the user-agent:

article, aside, details, div, dt, figcaption, footer, form, header, hgroup, html, main, nav, section, summary { display: block; }

In order to remove the effects of the flex-box environment, I isolate the div in the following manner:

<div> <div> <div> 123 </div> </div> </div>

The following box-model properties exist on the third division:

box-sizing: border-box display: block float: none line-height: 25.5px position: static z-index: auto

If we consider the qualities of this display mode:

[block]
<div> <div> <div style="background-color: magenta; color: #ffffff; padding: 2px 5px;"> [block] </div> </div> </div>

Compare them with the following display mode:

[inline-block]
<div> <div> <div style="display: inline-block; background-color: magenta; color: #ffffff; padding: 2px 5px;"> [inline-block] </div> </div> </div>

Modern browsers convert inline-block to block if the float property is added.

[inline-block -> block]
<div> <div> <div style="display: inline-block; float: left; background-color: magenta; color: #ffffff; padding: 2px 5px;"> [inline-block -> block] </div> </div> </div>

layout mode: block = 100% width
